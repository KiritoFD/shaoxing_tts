# 绍兴方言音标与吴拼书写说明

## 数据来源

本项目处理的是《绍兴方言研究》PDF 中的词条音标。页面通常按下面结构排版：

```text
汉字词条  音标/调值  释义
```

有些词条有多个读法，会在同一行或同一词条附近连续出现多段音标。PDF 中的音标很小，尤其是上标调值、鼻化符号和送气符号。

## Excel 标签现状

`result_all_converted.xlsx` 里的人工标注当前采用的是吴拼，因为输入简单。也就是说，第一版 OCR 训练目标不应直接设为国际音标，而应设为吴拼字符串；识别后再通过明确的“吴拼 -> 国际音标”对应关系转换。

当前关键列：

- `汉字`：词条。
- `拼音`：人工标注的吴拼，带调值数字。
- `IPA识别`：旧流程里的识别/转换结果，不能再当作第一训练目标。

## 吴拼调值规则

PDF 视觉上常有两类调值数字：

- 右下角数字。
- 右上角数字。

标注规则是：

1. 优先取右下角数字。
2. 如果右下角没有数字，再取右上角数字。

因此训练标签中的数字是按照这个规则整理后的吴拼调值，不要求 OCR 模型复刻上标/下标的位置。模型只输出普通文本数字即可。

## 第一版训练目标

第一阶段模型目标应改为：

```text
输入：裁剪后的原书音标图像
输出：NFC 归一化后的吴拼字符串，也就是 Excel 的 拼音 列
```

不输出：

- 汉字词条。
- 中文释义。
- PDF 标点。
- 上标/下标的视觉位置。

这样做的好处是字符集简单、标签可信，后续再用规则表把吴拼映射到国际音标。

## 旧 IPA 列说明

`IPA识别` 列目前有 5483 条非空记录。按字符统计，它不是完整保留 PDF 原排版的音标，而是更接近“归一化后的 IPA 音段序列”：

- 不含空格。
- 不含标点。
- 基本不含调值数字。
- 多个读法通常被拼接到同一字符串。

这列可以作为后处理或映射验证参考，但不作为第一版 OCR 训练标签。

## 已出现字符

当前标签字符集共 33 个字符：

```text
a b d f h i k l m n o p s t u v y z
ŋ ʔ ʰ ɦ ɕ ʑ ɒ ɤ æ ø ə ẽ ȵ ɡ ̃
```

其中：

- `ŋ`：软腭鼻音。
- `ʔ`：喉塞音。
- `ʰ`：送气标记，常跟在塞音或塞擦音后。
- `ɦ`：浊喉擦音或相关声母标记。
- `ɕ`、`ʑ`：龈腭擦音。
- `ȵ`：龈腭鼻音。
- `ɒ`、`ɤ`、`æ`、`ø`、`ə`、`ẽ`：元音或鼻化元音。
- `̃`：组合鼻化符号，可能和前一个元音组合显示。

## 需要统一的书写规范

训练前必须固定标签规范，否则模型会学习到互相冲突的目标。

### 1. 声调是否保留

PDF 原文中能看到 `11`、`33`、`52`、`55`、`335` 等调值，视觉上常以上标形式出现。当前 Excel 的 `IPA识别` 不保留这些调值。

建议：

- 第一版训练：不保留调值，复现 Excel 标签。
- 第二版训练：如果需要忠实转写 PDF，再另建 `IPA_with_tone` 标签列。

### 2. 多读法如何表示

当前标签里多读法常直接拼接，例如一个词条可能含两个读法但没有分隔符。模型训练时最好保持和 Excel 一致。

建议：

- 第一版：继续无分隔拼接，保证和现有标签兼容。
- 后续：可新增分隔符，如空格或 `|`，但必须重标所有多读法样本。

### 3. Unicode 归一化

鼻化可以写成预组合字符（如 `ẽ`），也可以写成基本元音加组合符（如 `e` + `̃`）。当前数据两种形式可能并存。

建议：

- 训练 manifest 使用 NFC 归一化。
- 字符表从归一化后的标签自动生成。
- 检查组合符 `̃` 是否悬空，即前面没有可鼻化元音。

### 4. 可疑字符

当前标签中普通字母 `l` 出现频率很高，部分样例看起来像旧 OCR 后处理遗留。训练前不强行删除，但需要在评估报告里统计：

- 标签长度异常。
- 重复字符异常，如 `lllll`。
- 不在音标字符表中的字符。
- 与 `拼音` 列长度明显不匹配的样本。

## 后续国际音标转换

因为吴拼和国际音标对应关系是清楚的，建议单独维护转换规则：

```text
吴拼 OCR 输出 -> 规则归一化 -> 国际音标
```

这样可以把“图像识别问题”和“音系转写问题”拆开，便于测试和纠错。
