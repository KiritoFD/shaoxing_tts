# 绍兴方言 IPA OCR 方案

## 结论

这个项目不适合只依赖通用中文 OCR。页面主体是“汉字词条 + IPA/声调符号 + 释义”，其中最难的是 IPA 和上标调值；通用 OCR 会把这些符号误读成拉丁字母、数字或乱码。成熟路线应当是：

1. 用 PaddleOCR、DBNet/PP-OCR 一类成熟检测器做行/词条定位。
2. 对 PDF 页面高 DPI 渲染并增强，稳定细笔画和声调号。
3. 对裁剪出的词条行做专用识别：优先 Calamari OCR/ocr-ipa 预训练模型，数据足够后再微调；备选 TrOCR 微调。
4. 用 Excel 里的词条顺序和页面号做结构校验，避免纯 OCR 自由输出。

## 成熟方案对比

| 方案 | 适合度 | 优点 | 风险 |
| --- | --- | --- | --- |
| PaddleOCR 检测 + Calamari/ocr-ipa 识别 | 推荐 | 已有 IPA OCR 项目可复用，适合行级 OCR，小样本微调成本低 | 需要准备逐行 IPA 标注 |
| PaddleOCR 识别模型自定义字典微调 | 可做 | 工程统一，中文检测/识别生态成熟 | IPA 和上标调值不一定适配默认识别头，训练配置较重 |
| TrOCR/Donut 类 Transformer 微调 | 备选 | 对复杂字符集可端到端学习，HuggingFace 工具链方便 | 小数据容易过拟合，推理较慢，需要较干净标注 |
| 只增强 PDF 后跑通用 OCR | 不推荐单独使用 | 工程简单，能改善检测和部分普通文字 | 不能从根本解决 IPA 字符集和声调号误识别 |

## 推荐实施步骤

### 第一阶段：先让输入稳定

用新脚本把指定页渲染为 400 DPI 增强图：

```powershell
py ipa_ocr_work\scripts\preprocess_pdf_pages.py --start-page 123 --end-page 130 --save-original
```

输出在 `ipa_ocr_work/output/enhanced_pages/`。默认做灰度去噪、CLAHE 对比增强、锐化和 adaptive threshold。这个阶段的目标不是直接得到最终 OCR，而是给检测器和标注工具提供更清晰的页面。

### 第二阶段：建立标注闭环

从增强图里按行裁剪词条，生成类似下面的训练清单：

```text
ipa_ocr_work/dataset/lines/page_123_001.png    læ̃¹¹sɔ¹¹ti³³die⁵²
```

先人工校 100-300 行，覆盖常见 IPA 字符、上标调值、鼻化符号和声母组合。Excel 里的 `汉字`、`页码` 列可以辅助匹配，但 IPA 仍需要人工或半自动校准。

### 第三阶段：模型路线

优先使用 Calamari/ocr-ipa：

- 输入是行图，和本项目“词条行里有一段 IPA”的版式接近。
- 字符集可控，适合这种异形字/特殊符号。
- 微调成本比从零训练低。

如果 Calamari 识别仍对上标调值不稳，再试 TrOCR 微调。TrOCR 的优势是端到端序列识别，但需要更多标注和更严的验证集。

### 第四阶段：合并和校验

OCR 输出不要直接覆盖 Excel。建议新增列：

- `IPA_OCR_raw`
- `IPA_OCR_norm`
- `IPA_OCR_conf`
- `IPA_review_status`

用规则检查非法字符、声调数字缺失、词条数不一致、同页顺序错位。低置信度行进人工复核队列。

## 下一步

最小可行版本是：增强 123-130 页，抽取 200 行裁剪图，手工校一个 TSV，然后用 ocr-ipa/Calamari 微调或先做零样本评估。评估指标用 CER 加“完全匹配率”，不要只看肉眼样例。
